# Чеклист Выполнения Этапа

Этот чеклист является gate для работы по этапам. Если пункт не выполнен, этап нельзя считать начатым или закрытым.

## Preflight Перед Этапом

- [ ] Проверен `planning/technical-debt.md`.
- [ ] Блокеры техдолга отсутствуют или явно включены в scope этапа.
- [ ] Проверены необходимые инструменты, команды запуска и тесты.
- [ ] Проверен `planning/feature-requests.md`.
- [ ] Фиче-реквесты, влияющие на этап, подсвечены и согласованы.
- [ ] Краткий план этапа с декомпозицией показан пользователю.
- [ ] Получено подтверждение плана.
- [ ] Создана рабочая ветка этапа.

## Готовность Среза К Проверке

- [ ] Реализация выполнена в ветке этапа.
- [ ] Документация обновлена под фактическое состояние проекта.
- [ ] `planning/mvp-roadmap.md` актуализирован.
- [ ] `planning/technical-debt.md` актуализирован.
- [ ] `planning/feature-requests.md` актуализирован, если появились новые фичи-кандидаты.
- [ ] Выполнены необходимые тесты.
- [ ] Выполнена сборка frontend, если затронут UI.
- [ ] Выполнен smoke-тест, если этап меняет исполняемый сценарий.
- [ ] Изменения закоммичены в ветке этапа.
- [ ] Пользователю передан итог среза и список проверок.
- [ ] Получено подтверждение, что срез работает.

## Closeout После Подтверждения

- [ ] Ветка этапа смержена в `master`.
- [ ] Проверен `git status` после merge.
- [ ] Пользователю передан полный closeout этапа:
  - [ ] что удалось;
  - [ ] что не получилось или осталось ограничением;
  - [ ] какие проверки выполнены;
  - [ ] какие документы изменились;
  - [ ] какие пункты техдолга добавлены или закрыты;
  - [ ] какие фиче-реквесты добавлены или изменены;
  - [ ] нужны ли ревью тестов, архитектуры или процесса разработки.

Этап закрыт только после полного closeout после merge. Ответ вида "смержено" не закрывает этап.
