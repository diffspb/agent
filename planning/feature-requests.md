# Фиче-Реквесты

Этот файл отслеживает возможные фичи вне ближайшего MVP.

## Фичи-Кандидаты

### FR-001: Создание Запроса На Слияние

Статус: Кандидат

Позволить агенту создавать ветку, коммитить изменения, пушить ветку и открывать запрос на слияние после завершения задачи.

### FR-002: Продуктовый Веб-Интерфейс Для Запусков И Событий

Статус: Кандидат

Добавить полноценный продуктовый веб-интерфейс для запусков задач, вызовов инструментов, ошибок и статистики.

Примечание: тестовый React/Vite UI для проверки REST API входит в MVP. Этот фиче-реквест относится к полноценному интерфейсу, который будет полезен после MVP.

### FR-003: Контейнеризованные Рабочие Пространства

Статус: Кандидат

Запускать рабочие пространства задач внутри контейнеров, чтобы улучшить изоляцию и воспроизводимость.

### FR-004: Параллельное Выполнение Несколькими Рабочими Процессами

Статус: Кандидат

Разрешить нескольким рабочим процессам обрабатывать задачи параллельно.

### FR-005: Точки Ручного Подтверждения

Статус: Кандидат

Требовать явное подтверждение для рискованных вызовов инструментов, обновлений зависимостей, изменений с влиянием на продакшен или крупных рефакторингов.

### FR-006: Долгосрочная Память О Проекте

Статус: Кандидат

Сохранять переиспользуемые знания о проекте между задачами: архитектурные заметки, типовые команды и повторяющиеся решения реализации.

### FR-007: Интеграция С Реальным Таск-Трекером Через MCP

Статус: Кандидат

Подключить реальный таск-трекер через MCP или реализовать production-ready MCP-сервер для выбранного трекера.

Примечание: примитивный MCP-эмулятор таск-трекера входит в MVP. Этот фиче-реквест относится к интеграции с реальной внешней системой.
