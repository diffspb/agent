# Политика Безопасности Инструментов

## Назначение

Этот документ фиксирует минимальную MVP-политику для локальных инструментов агента. Политика нужна, чтобы инструменты этапа 5 не выполняли операции за пределами рабочего пространства run.

## Рабочее Пространство

Рабочие пространства создаются в `WORKSPACE_ROOT`, по умолчанию `.data/workspaces`.

Если задан `PROJECT_REPO_ROOT`, каталог `repo/` подготавливается как `git worktree` рабочей ветки задачи. Если переменная не задана, `repo/` создается как обычный локальный каталог для изолированных тестовых сценариев.

Структура run:

```text
run-<id>-<task-id>/
  repo/
  artifacts/
  logs/
  scratch/
```

Файловые и shell-инструменты по умолчанию работают внутри `repo/`.

## Ограничения Путей

- Инструменты принимают только относительные пути.
- Абсолютные пути запрещены.
- Переходы за пределы workspace через `..` запрещены.
- Ошибки path traversal возвращаются как ошибка tool.

## Shell-Команды

`run_command` запускает команды без `shell=True`: команда может передаваться списком аргументов или строкой, которая разбирается через `shlex.split`.

Запрещенные команды MVP:

```text
chmod
chown
dd
mkfs
mount
reboot
rm
shutdown
sudo
umount
```

Тайм-аут по умолчанию: `TOOL_COMMAND_TIMEOUT_SECONDS=10`.

## Лимиты Вывода

- `TOOL_OUTPUT_MAX_BYTES`: максимум сохраняемого stdout/stderr для shell tools.
- `TOOL_FILE_READ_MAX_BYTES`: максимум читаемого содержимого файла.

Если лимит превышен, результат обрезается и помечается флагом `truncated`.

## Логирование

Каждый вызов инструмента записывается в `tool_calls`:

- `run_id`;
- имя tool;
- входной JSON;
- статус;
- выходной JSON;
- ошибка;
- время начала и завершения.

## Ограничения MVP

Политика не является production sandbox. Она не заменяет контейнеры, системные sandbox-механизмы или ручные approvals для рискованных операций. Усиление политики вынесено в технический долг.
