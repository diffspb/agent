# 0002. Ограничения Разработки И Локального Запуска

Дата: 2026-05-01
Статус: Предложено

## Контекст

Проект должен развиваться постепенно, с проверяемыми этапами MVP. Важно заранее зафиксировать правила работы с ветками, ограничения для автономных агентов, требования к виртуальному окружению Python и сценарии запуска.

Сервис целится в Docker-использование, но локальная разработка должна оставаться возможной без Docker.

## Решение

Принять следующие ограничения:

- Разработка функциональности ведется в отдельных ветках.
- Merge в `master` выполняется после завершения этапа или задачи и подтверждения, что все работает.
- В `master` допускаются прямые изменения документации и небольшие фиксы.
- Агентам, запущенным в фоне или автономно, запрещено пушить в `master`.
- Python-зависимости устанавливаются только в виртуальное окружение.
- Python-тесты запускаются только в виртуальном окружении.
- Для Python-зависимостей используется `pip`.
- Для активации окружения можно использовать алиас `vactivate`.
- Локальный запуск без Docker должен поддерживаться наравне с Docker-сценарием разработки.
- Тестовый frontend хранится в `frontend/`, делается на React и запускается через Vite.
- Для локальной разработки нужен примитивный MCP-эмулятор таск-трекера.

## Последствия

Положительные:

- Снижается риск случайных изменений в `master` от автономных запусков.
- Каждый этап можно проверять до merge.
- Локальная разработка остается быстрой и не зависит от Docker.
- MCP-интеграцию можно тестировать до подключения реального таск-трекера.
- REST API можно проверять через небольшой веб-интерфейс без написания curl-команд вручную.

Отрицательные:

- Нужно поддерживать два сценария запуска: локальный и Docker.
- Появляется дополнительный компонент MVP — MCP-эмулятор таск-трекера.
- Frontend увеличивает объем MVP, даже если он нужен только для тестирования.

## Рассмотренные Альтернативы

### Делать Только Docker-Запуск

Отклонено. Docker остается целевым способом упаковки, но локальная разработка без Docker нужна для скорости.

### Не Делать UI В MVP

Отклонено. Небольшой UI нужен как удобный инструмент проверки REST API и демонстрации полного сценария.

### Подключать Сразу Реальный Таск-Трекер

Отклонено для MVP. MCP-эмулятор позволяет независимо разработать и проверить агентский цикл.
