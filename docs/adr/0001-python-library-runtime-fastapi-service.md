# 0001. Исполнительное Ядро Как Python-Библиотека И FastAPI Как Управляющий Сервис

Дата: 2026-05-01
Статус: Предложено

## Контекст

Проекту нужен автономный агент для написания кода, управляемый через таск-трекер. Агент должен использовать MCP для взаимодействия с трекером и предоставлять REST API для инициации задач, просмотра истории попыток выполнения и статистики.

Ключевое поведение агента не должно быть связано с обработкой HTTP-запросов. Один и тот же runtime должен переиспользоваться из тестов, CLI-команд, рабочих процессов или FastAPI-сервиса.

## Решение

Реализовать агента как Python-библиотеку и предоставить операционное управление через тонкий FastAPI-сервис.

Библиотека будет содержать:

- Runtime агента.
- Контроллер агента.
- Lifecycle выполнения задачи и policy layer.
- Реестр инструментов.
- Адаптер таск-трекера.
- Менеджер рабочих пространств.
- Интерфейсы наблюдаемости.
- Интерфейс LLM-клиента.

FastAPI будет содержать:

- Схемы запросов и ответов.
- Управляющие маршруты.
- Сборка зависимостей.
- Эндпоинты healthcheck, истории попыток выполнения и статистики.
- Точки подключения для фонового выполнения задач.

## Последствия

Положительные:

- Runtime можно тестировать без HTTP.
- Сервисный слой остается тонким.
- Будущие CLI или точки входа рабочих процессов смогут использовать тот же runtime.
- Адаптеры таск-трекера, LLM и хранилища смогут развиваться независимо.

Отрицательные:

- На старте нужно больше явных интерфейсов.
- Проекту нужна сборка зависимостей вместо прямых вызовов классов реализации из обработчиков маршрутов.

## Рассмотренные Альтернативы

### Поместить Всю Логику В Обработчики FastAPI

Отклонено на текущем этапе. Это ускорило бы прототипирование, но связало бы бизнес-логику с HTTP и усложнило бы фоновую работу, тестирование и будущие рабочие процессы.

### Сначала Сделать Отдельный CLI

Отложено. CLI может быть полезен позже, но продуктовое требование уже включает REST API для инициации задач и статистики.
