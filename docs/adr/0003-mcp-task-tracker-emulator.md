# 0003. MCP-Эмулятор Таск-Трекера С JSON-Состоянием

Дата: 2026-05-01
Статус: Принято

## Контекст

Для MVP нужен таск-трекер, с которым агент сможет взаимодействовать через MCP. Подключение реального таск-трекера на первом этапе увеличит сложность и замедлит разработку агентского цикла.

Нужно иметь возможность запускать разные сценарии задач и продолжать работу с состояния, полученного в предыдущем запуске.

## Решение

Реализовать примитивный MCP-эмулятор таск-трекера для одного проекта.

Эмулятор:

- при запуске читает состояние из указанного JSON-файла;
- во время работы хранит состояние в памяти;
- при изменениях сбрасывает актуальное состояние во временный JSON-файл;
- использует email как идентификатор пользователя;
- поддерживает фиксированный workflow `Todo`, `Open`, `InProgress`, `InReview`, `NeedsInfo`, `Done`, `Cancelled`;
- разрешает любые переходы между статусами;
- поддерживает задачи типов `epic`, `task`, `test`;
- поддерживает связи `part_of_epic`, `blocks`, `blocked_by`;
- предоставляет MCP API для получения задачи, изменения задачи, списка задач по статусу и исполнителю, комментариев и описания workflow.

## Последствия

Положительные:

- Можно разрабатывать агентский цикл без реального таск-трекера.
- Можно готовить JSON-сценарии для разных тестов.
- Можно продолжать работу с состояния, созданного предыдущим запуском.
- MCP-интеграция становится частью MVP с ранних этапов.

Отрицательные:

- Эмулятор не покрывает сложность реального таск-трекера.
- Нужно поддерживать формат JSON-сценариев.
- Поведение workflow намеренно упрощено: любые переходы разрешены.

## Рассмотренные Альтернативы

### Хранить Состояние Эмулятора В SQLite

Отклонено для MVP. JSON-файлы проще использовать как сценарии и проще переносить между тестами.

### Сразу Подключить Реальный Таск-Трекер

Отклонено для MVP. Реальная интеграция будет отдельной задачей после стабилизации агентского цикла.
